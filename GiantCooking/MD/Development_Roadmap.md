# 『アンチグラビティ（仮）』開発ロードマップ & タスクリスト (Meta Quest対応版)

本ドキュメントは、Meta Questでの動作を前提としたVRゲーム『アンチグラビティ（仮）』の開発工程を定義したものです。

---

## 1. 開発フェーズ一覧

### フェーズ0：VR基盤セットアップ (Meta Quest)
Meta Questで正しく動作し、操作できる環境を構築します。
- [ ] 0.1 **XR Originの設定**：XR Interaction Toolkitを用いたプレイヤーリグの配置。
- [ ] 0.2 **コントローラー・ハンドのセットアップ**：Questコントローラーおよびハンドトラッキングの入力設定。
- [ ] 0.3 **ビルド設定の確認**：Androidプラットフォーム、OpenXR(Meta Quest)の設定確認。

### フェーズ1：コア・物理剣戟システム (VR版)
VR空間で「剣を持ち、弾き合う」手応えを構築します。
- [ ] 1.1 **XR Grab Interactableの設定**：剣を両手または片手で掴めるようにし、物理追従（Velocity Tracking）を有効化。
- [ ] 1.2 **物理衝突によるガード判定**：剣同士が物理的にぶつかったことを検知し、攻撃を相殺するロジックの実装。
- [ ] 1.3 **物理はじきモーション**：剣同士が触れた際、物理演算でコントローラーの手が弾かれるような視覚・物理的フィードバック。
- [ ] 1.4 **VR無敵時間演出**：弾かれた際、視覚的なエフェクト（剣の明滅等）と共に無敵状態へ移行。

### フェーズ2：『極（きわみ）ゲージ』システム
VR内での視認性と直感的な必殺技発動を実装します。
- [ ] 2.1 **VR用ゲージUI (Diegetic UI)**：剣の柄や手首に浮かぶ、VR空間に溶け込んだゲージの作成。
- [ ] 2.2 **接触時のゲージ増加ロジック**：攻防に応じたゲージ加算。
- [ ] 2.3 **「一閃」アクション (Quest操作対応)**：ゲージMAX時にトリガーボタンを深く引く、あるいは特定のジェスチャーで発動。

### フェーズ3：フィールドと勝利条件
VR酔いに配慮しつつ、迫力ある場外落下を実装します。
- [ ] 3.1 **VR向け浮遊ステージ**：プレイヤーの足元を意識させるテクスチャや環境構築。
- [ ] 3.2 **落下判定とフェード演出**：落下時に暗転（フェード）させることで、VR酔いを防止しつつリスタート。
- [ ] 3.3 **リスタート・キャリブレーション**：再開時にプレイヤーの位置と向きを適切にリセット。

### フェーズ4：UX/演出 (Meta Quest向け最適化)
Questのハードウェア特性を活かした演出。
- [ ] 4.1 **XRI Haptic Feedback**：掴んでいる手に直接響く、Meta Questコントローラーの振動制御。
- [ ] 4.2 **3Dオーディオ (Spatialize)**：剣の衝突位置から音が聞こえる空間音響の設定。
- [ ] 4.3 **Quest向け軽量VFX**：モバイル環境で負荷の低い火花やオーラ演出。
